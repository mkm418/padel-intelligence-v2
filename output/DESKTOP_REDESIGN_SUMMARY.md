# PadelPassport Desktop Redesign Specification

## Executive Summary

Transform PadelPassport from a mobile-first app into a **premium sports analytics platform** optimized for desktop (1200px+). The current design suffers from cramped layouts, inappropriate mobile patterns (bottom tab bar), and underutilized screen real estate.

**Target Aesthetic**: Strava Dashboard meets ESPN Stats - data-rich, information-dense, professional sports analytics.

---

## Current Issues Identified

| Issue | Current State | Desktop Solution |
|-------|--------------|------------------|
| Navigation | Bottom tab bar (mobile pattern) | Fixed left sidebar with collapsible sections |
| Layout | Single-column, centered, narrow | Multi-column grids (2-4 columns) |
| Cards | Stacked vertically, narrow | Horizontal cards, larger grids |
| Modals | Full-screen overlays | Centered modals (400-1000px width) |
| Density | Low information density | Data-rich tables and dashboards |
| Hover | No hover states | Rich hover interactions |

## 9. Interactive States

### Hover Effects
```css
/* Cards */
.card:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.4);
  border-color: rgba(255, 255, 255, 0.1);
}

/* Table rows */
.table-row:hover {
  background: rgba(255, 255, 255, 0.04);
}

/* Buttons */
.btn:hover {
  transform: translateY(-1px);
  filter: brightness(1.05);
}
```

### Focus States
```css
:focus-visible {
  outline: 2px solid var(--brand-primary);
  outline-offset: 2px;
}
```

## Migration Strategy

Since the current app uses inline styles in React components, migration should:

1. Create a Tailwind config with the design tokens
2. Build reusable components (Card, Modal, Table, etc.)
3. Add desktop-specific layouts wrapped in media queries
4. Keep mobile layouts as fallback for < 1200px
5. Progressive enhancement: mobile-first stays, desktop adds
